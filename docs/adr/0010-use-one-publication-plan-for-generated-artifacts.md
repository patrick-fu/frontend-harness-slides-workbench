# Use one Publication Plan for generated artifacts

Derive Manifest data, dynamic statistics, preview destinations, capture
targets, and browser-audit cases from one deterministic, model-neutral
Publication Plan. Filesystem, browser, encoder, and command-line work remain
adapters around that pure projection instead of inventing their own order or
selection rules.

Preview capture is manual and explicit. Partial capture touches only selected
Topic-ID WebPs; ordinary builds, validation, Model ID changes, and Manifest
generation never capture or clean previews.
